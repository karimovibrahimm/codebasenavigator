# Codebase Navigator

A demo web app that helps developers quickly understand an unfamiliar codebase.
Pick a project folder and it reads the source, summarizes the project, maps the
architecture, explains individual files, and answers questions in a chat — a small
demonstration of a *persistent codebase understanding* experience.

> MVP demo. Structure analysis is heuristic; the AI prose is generated by an LLM.
> Some details are intentionally simplified — the goal is the experience, not a
> perfect static analyzer.

## Features

- **Project upload** — select a local folder in the browser. `node_modules`,
  build output, lockfiles and binaries are filtered out automatically. Files are
  read entirely client-side.
- **Import from GitHub** — paste a public repo URL (e.g. `github.com/owner/repo`,
  optionally a `/tree/branch` URL). The app fetches the source server-side via the
  GitHub API + raw content, applying the same filters as folder upload. Set
  `GITHUB_TOKEN` to raise the rate limit.
- **AI overview** — project name, framework, languages, file count, an estimated
  architecture, and a plain-English summary of what the project does.
- **File explorer + viewer** — collapsible tree; click a file to see its code plus
  AI-generated purpose & responsibilities, parsed exports, and its relationships.
- **Architecture view** — files grouped into layers (Frontend, Backend, API,
  Authentication, Database, Utilities) with an AI explanation per layer.
- **Relationship view** — for each file, **Depends on** / **Used by**, computed
  from a local import graph and clickable to navigate.
- **AI chat** — ask questions like "How does authentication work?". Answers are
  grounded in the most relevant files (keyword retrieval) and cite their sources.
- **Export for AI** — bundle the whole project into a single Markdown file (an
  ASCII file map + every file's contents) and **copy** it or **download** it to
  paste into Cursor, Claude, or any LLM. A "map only" option exports just the
  tree. Works for both folder uploads and GitHub imports.

## Tech stack

- **Next.js (App Router) + TypeScript** — UI and API routes in one app
- **Tailwind CSS** — dark, three-column layout (Explorer · Center · Chat)
- **Groq** (`llama-3.3-70b-versatile` by default) for all AI prose, called only
  from server-side API routes so the key is never exposed to the browser

## Getting started

```bash
npm install
```

Add your Groq API key (free at <https://console.groq.com/keys>) to `.env.local`:

```
GROQ_API_KEY=your_key_here
GROQ_MODEL=llama-3.3-70b-versatile
```

Then run the dev server:

```bash
npm run dev
```

Open <http://localhost:3000>, then either **select a project folder** or click
**Try a sample project** to load the bundled example instantly.

## How it works

| Concern | Approach |
| --- | --- |
| File reading | Browser `webkitdirectory` picker, filtered + size-capped client-side |
| Framework / languages | Heuristics over `package.json`, extensions, signature files |
| Import graph | Regex-based resolution of relative imports (no AST) |
| Architecture layers | Keyword/path bucketing into a fixed set of categories |
| Chat retrieval | Keyword ranking of files (no embeddings) |
| AI prose | Groq chat completions via server-side API routes |

## Project structure

```
app/
  api/            overview · file · architecture · chat · health routes
  page.tsx        renders the app shell
components/       Provider + Explorer / Center / Chat panels and views
lib/              parsing, ignore rules, framework detection, import graph,
                  retrieval, Groq client, and the bundled sample project
```

## Limitations

- In-memory, single session — no persistence or auth.
- Best suited to demo-sized repos; large projects are capped for speed/tokens.
- Import resolution and architecture bucketing are heuristic, not exhaustive.
